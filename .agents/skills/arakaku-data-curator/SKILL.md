# Unofficial ARAKAKU Database Data Curator Skill

Use this skill when the task changes canonical project data.

Canonical data lives in:

```text
data-src/*.csv
```

Generated JSON lives in:

```text
docs/data/*.json
```

Do not edit generated JSON directly.

---

## Core rules

1. Edit CSV, not JSON.
2. Do not invent confirmed bout results.
3. Keep uncertain extracted data in `review/`.
4. Run validation after changes.
5. Remove generated JSON after checks.
6. Know which CSVs are hand-edited vs regenerated before editing (see "CSV provenance").

Required final checks:

```bash
make check
make clean-generated
```

---

## CSV provenance

Not every `data-src/*.csv` is hand-editable. Some are regenerated by scripts and will lose hand edits on the next rebuild. This must match the provenance table in `README.md` (the authority is the Makefile `clean-stage1-csvs` / `clean-stage2-csvs` targets plus each generator script).

**canonical (hand-edit these; no generator, kept from git on rebuild):**

- `events.csv`
- `bouts.csv`
- `bout_participants.csv`
- `fighters.csv`
- `fighter_snapshots.csv`
- `videos.csv`
- `title_reigns.csv`

**stage1 generated (raw ingest; `make generate-stage1`):**

- `numbers_fighters.csv` / `numbers_name_matches.csv` / `numbers_fight_records.csv` (`extract_numbers.py`)
- `archives/youtube.csv` / `archives/note.csv` (`archive_metadata.py`)
- `articles.csv` (`generate_articles.py`)
- `source_documents.csv` / `source_mentions.csv` (`build_source_documents.py`)
- `official_players.csv` / `official_tournaments.csv` / `official_matches.csv` / `official_history.csv` (`generate_official_csvs.py`)

**stage2 generated (derived; `make generate-stage2`):**

- `promotions.csv` (`generate_promotions.py`)
- `titles.csv` (`generate_titles.py`)
- `aliases.csv` (`generate_aliases.py`)
- `video_links.csv` (`generate_video_links.py`)
- `article_links.csv` (`generate_article_links.py`)
- `official_news.csv` / `official_pages.csv` (`generate_official_pages_csv.py`)

> Hand edits are lost on regeneration: the factual data for `promotions.csv` and `titles.csv` lives in the `PROMOTIONS` / `TITLES` Python literals inside `scripts/generate_promotions.py` and `scripts/generate_titles.py`, not in the CSVs. Editing those CSVs directly is overwritten by `make rebuild` / `make rebuild-stage2` / `make regenerate-csvs`. To fix a promotion or title fact, edit the generator literal.

## Canonical CSVs

### promotions.csv

Promotion and organization data.

Use this for:

- promotion name
- category
- rules
- summary

Stage2 generated. The facts live in the `PROMOTIONS` literal in `scripts/generate_promotions.py`. Edit that literal, not the CSV (CSV edits are overwritten on rebuild).

### events.csv

Event data.

`event_id` connects:

- bouts
- videos
- source mentions
- viewer event pages

Do not create duplicate event IDs.

### bouts.csv

Bout data.

Important distinction:

- bout-level facts live here
- `result_status=unknown`: use when result is not confirmed
- method / round / time are confirmed result details only

Do not derive winner/loser from matchup order.

### bout_participants.csv

Bout participant relationship data.

Use this for:

- `bout_id` to `bouts.csv`
- `fighter_id` to `fighters.csv` when identified
- preserved `fighter_name`
- participant `result` such as `win`, `loss`, or `unknown`

`matchup`, `winner`, and `loser` are generated JSON display fields, not source columns.

### fighters.csv

Fighter data.

Use stable `fighter_id` values. Be careful with same-name and similar-name fighters.

### Numbers-derived CSVs

Numbers-derived comparison data lives in:

```text
numbers_fighters.csv
numbers_name_matches.csv
numbers_fight_records.csv
```

These are generated from `data-raw/アラカク選手名鑑.numbers` by `scripts/extract_numbers.py`.

Use them as comparison and review inputs:

- `numbers_fighters.csv`: raw-ish profile rows from the Numbers "全体" sheet
- `numbers_name_matches.csv`: inferred name-to-fighter matches and generated candidate IDs
- `numbers_fight_records.csv`: personal fight records from the Numbers "個人成績" sheet

Do not directly promote Numbers fight records into `bouts.csv` or `bout_participants.csv`. Pairing, duplicate detection, and contradiction checks should happen in viewer JavaScript or a review workflow first.

### videos.csv

Video metadata. Canonical (hand-edit).

Do not duplicate videos unless URLs differ or there is a clear reason.

`scripts/import_youtube_videos.py` can append yt-dlp rows, but by default it preserves curated values: it only fills blank cells on existing rows and adds new rows, never overwriting non-empty fields. Use `--replace` only when you intend to discard curated edits.

### article_links.csv

Link articles to entities.

Use this instead of comma-list source article columns on entity tables.

### video_links.csv

Link videos to entities.

Allowed `entity_type` values:

```text
event
bout
fighter
promotion
title
```

Common `relation_type` values:

```text
full_fight
highlight
short
stream_archive
preview
interview
commentary
reference
```

### titles.csv

Title and tournament entity data.

Use this for title ID, promotion, and division.

Stage2 generated. The facts live in the `TITLES` literal in `scripts/generate_titles.py`. Edit that literal, not the CSV (CSV edits are overwritten on rebuild).

### title_reigns.csv

Title and tournament lineage data.

Use this for reign order, fighter, won/lost event references, and source article/video references.

### source_documents.csv

Full-text source database for:

```text
note_article
youtube_description
```

Usually generated by `scripts/build_source_documents.py`.

### source_mentions.csv

Extracted mentions from source text.

Treat as candidates, not confirmed facts.

Common `mention_type` values:

```text
event
matchup
result
note_url
youtube_url
```

### archives/youtube.csv and archives/note.csv

Archive metadata generated from local caches:

```text
archives/youtube.csv
archives/note.csv
```

These CSVs are committed source metadata even though `tmp/` cache files are not committed.

Use them for:

- viewer display enrichment
- search enrichment
- preserving external title/description metadata when remote pages change

Do not use archive rows to confirm bout winners, methods, fighter identities, title lineage, or other canonical facts without checking source context.

## Data update workflow

For direct CSV edits:

```bash
make check
make clean-generated
```

For source document refresh:

```bash
make cache-sources
make archive-metadata
make build-sources
make check
make clean-generated
```

For large changes:

1. Create or update candidate CSVs under `review/`.
2. Inspect counts and confidence.
3. Confirm entity IDs.
4. Apply reviewed changes to `data-src/`.
5. Run `make check`.
6. Run `make clean-generated`.

---

## Never do these

Do not:

- edit `docs/data/*.json` directly
- commit `docs/data/*.json`
- commit `tmp/note-html/*.html`
- commit `tmp/youtube-info/*.info.json`
- infer winner/loser from title, order, or vague context
- bulk-apply extracted rows without review
