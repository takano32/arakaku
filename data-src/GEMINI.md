# ARAKAKU Data Instructions

Canonical data for ARAKAKU is stored in this directory as CSV files.

## CSV Maintenance Rules

### 1. IDs are Immutable
Once assigned, `event_id`, `fighter_id`, `bout_id`, etc., should not be changed without a very strong reason, as they are used for cross-referencing.

### 2. Mandatory Fields
Ensure all mandatory fields (as defined in `scripts/validate_json.py`) are present.
- `event_id` is required for bouts and video links.
- Bout participants are stored in `bout_participants.csv`; do not add embedded fighter columns to `bouts.csv`.

### 3. Data Entry Conservatism
- If a result is not explicitly confirmed by a source, set `result_status=unknown`.
- Do not infer winners from matchup order.
- Use participant `result` values in `bout_participants.csv` only when the result is confirmed.
- Numbers-derived fight records are review/comparison data; do not directly convert them into confirmed bouts. The viewer will automatically supplement `unknown` results using this data.

## CSV Provenance (edit the right place)

Not every CSV here is hand-editable. Some are regenerated by scripts and lose hand edits on the next `make rebuild` / `make regenerate-csvs`. This mirrors the provenance table in the project `README.md`; the authority is the Makefile `clean-stage1-csvs` / `clean-stage2-csvs` targets plus each generator.

- **canonical (hand-edit these):** `events.csv`, `bouts.csv`, `bout_participants.csv`, `fighters.csv`, `fighter_snapshots.csv`, `videos.csv`, `title_reigns.csv`.
- **stage1 generated (`make generate-stage1`):** `numbers_fighters.csv`, `numbers_name_matches.csv`, `numbers_fight_records.csv`, `archives/youtube.csv`, `archives/note.csv`, `articles.csv`, `source_documents.csv`, `source_mentions.csv`, `official_players.csv`, `official_tournaments.csv`, `official_matches.csv`, `official_history.csv`.
- **stage2 generated (`make generate-stage2`):** `promotions.csv`, `titles.csv`, `aliases.csv`, `video_links.csv`, `article_links.csv`, `official_news.csv`, `official_pages.csv`.

Hand edits are lost on regeneration: the factual data for `promotions.csv` and `titles.csv` lives in the `PROMOTIONS` / `TITLES` Python literals inside `scripts/generate_promotions.py` and `scripts/generate_titles.py`, not in the CSVs. Fix promotion or title facts in those generator literals.

`videos.csv` is canonical, but `scripts/import_youtube_videos.py` can append yt-dlp rows. By default it preserves curated values (fills blank cells and adds new rows only, never overwriting non-empty fields). Use `--replace` only to discard curated edits.

## Key Files

- `promotions.csv`: Organizations and their meta-data. Stage2 generated from the `PROMOTIONS` literal in `scripts/generate_promotions.py`. Do not hand-edit the CSV.
- `events.csv`: Tournament and event details. `event_id` is the primary key. Canonical.
- `bouts.csv`: Individual matches. Reference fighters by name/ID. Canonical.
- `bout_participants.csv`: Bout-to-fighter participant rows and participant-level results. Canonical.
- `fighters.csv`: Fighter profiles and IDs. Canonical.
- `videos.csv`: Metadata for YouTube and other video sources. Canonical.
- `title_reigns.csv`: Title reign / tournament lineage rows. Canonical.
- `fighter_snapshots.csv`: Per-event/point-in-time fighter state. Canonical.
- `titles.csv`: Title and tournament entities. Stage2 generated from the `TITLES` literal in `scripts/generate_titles.py`. Do not hand-edit the CSV.
- `aliases.csv`: Name variants and aliases. Stage2 generated by `scripts/generate_aliases.py`.
- `video_links.csv`: Video-to-entity link rows. Stage2 generated by `scripts/generate_video_links.py`.
- `article_links.csv`: Article-to-entity link rows. Stage2 generated by `scripts/generate_article_links.py`.
- `articles.csv`: note article ledger. Stage1 generated by `scripts/generate_articles.py`.
- `numbers_fighters.csv`: Fighter profiles extracted from `data-raw/アラカク選手名鑑.numbers`. Stage1 generated by `scripts/extract_numbers.py`.
- `numbers_name_matches.csv`: Inferred links between Numbers fighter rows and canonical fighter IDs. Stage1 generated by `scripts/extract_numbers.py`.
- `numbers_fight_records.csv`: Personal fight-record rows from the Numbers "個人成績" sheet. Stage1 generated by `scripts/extract_numbers.py`.
- `archives/youtube.csv`: YouTube metadata extracted from local yt-dlp info JSON caches. Stage1 generated by `scripts/archive_metadata.py`.
- `archives/note.csv`: note metadata extracted from local HTML caches. Stage1 generated by `scripts/archive_metadata.py`.
- `source_documents.csv`: Full text database for verification. Stage1 generated by `scripts/build_source_documents.py`.
- `source_mentions.csv`: Extracted entity mentions for review. Stage1 generated by `scripts/build_source_documents.py`.
- `official_players.csv`: Player profiles from official site (`tmp/arakaku-site/data/players.json`). Stage1 generated by `scripts/generate_official_csvs.py`. Do not edit manually.
- `official_tournaments.csv`: Tournament results from official site. Stage1 generated by `scripts/generate_official_csvs.py`.
- `official_matches.csv`: Match records from official site. Stage1 generated by `scripts/generate_official_csvs.py`.
- `official_history.csv`: History entries from official site (flattened). Stage1 generated by `scripts/generate_official_csvs.py`.
- `official_news.csv`: News entries from official site. Stage2 generated by `scripts/generate_official_pages_csv.py`.
- `official_pages.csv`: Static pages from official site. Stage2 generated by `scripts/generate_official_pages_csv.py`.

## Workflow for New Data
1. Check `../review/` for existing candidates (`note_structured_results.csv`, `structured_result_patch_candidates.csv`, `youtube_description_candidates.csv`, `source_mention_result_candidates.csv`, `source_event_reference_candidates.csv`, `source_bout_reference_candidates.csv`, `source_video_reference_candidates.csv`).
2. Add new rows to the relevant canonical CSV (for generated CSVs, edit the generator instead).
3. Run `make check` from the project root to validate.
